前言

HTML是構成網頁的基礎架構，無論是透過JS創造的標籤或使用template語法最後都會被瀏覽器編譯成HTML執行渲染，HTML是一種標籤語法，除了部分特殊的標籤如:input、hr等以外其他語法在撰寫上都必須要有對應的標籤</>作為封閉。

HTML的結構不僅是控制css的要點之一，也是讓搜尋引擎更容易針對網頁進行資料蒐集的重點，如何正確使用語意化標籤在SEO上會有一定的影響，開始前建議先安裝vs code套件HTML Snippets可以減少撰寫HTML標籤時間。

執行網頁

使用vs code開啟資料夾後建立一個.html的檔案(首頁預設index.html)，使用emmet輔助工具提供的快捷語法只要輸入「!」一個驚嘆號就可以建立初始化網頁內容，使用瀏覽器執行檔案就會自動將檔案轉換成網頁語法呈現在畫面上，以下用tag代替「標籤」一詞介紹html的使用方式。

a tag

a tag是超連結的使用方式

  1. <a href="導向連結"></a>
    可選擇參數target(開啟新視窗方式)、title(提示文字)等

標題系列tag

文章標題皆建議使用此系列標籤，一般預設由大至小分別會有不同的粗體及字體大小

  1. <h1>內容</h1>
    <h2>內容</h2>
    ...
    <h6>內容</h6>

section tag

section語意為一個獨立區域的部分，一個section內應有標題的結構

  1. <section>
    其他標籤及內容
    <section>

article tag

article語意與section類似，一個article內應有標題、內文或圖片等相關結構

  1. <article>
    其他標籤及內容
    </article>

div tag

div語意為一個沒有特別含意的區域，在有article、section等情況下，div的使用時間應要減少

  1. <div>其他標籤或內文</div>

span tag

span語意為行內的元素，一般如果用於句子中某部分內容時會套用其他樣式

  1. <p>內文內文內文<span>套用其他樣式的內文</span></p>

ol tag

ol語意為有序清單，其本身自帶list-style:decimal的屬性，有各種參數可以使用

  1. <ol>
      <li>內文</li>
      <li>內文</li>
    </ol>

ul tag

ul語意為無序清單，當子內容為同類型項目時父元素可以選擇用ul包覆，表明這個ul內是一組清單

  1. <ul>
      <li>內文</li>
      <li>內文</li>
    </ul>

main tag

main語意為網頁中主要內容區域，使用main在無障礙網頁中會有提升體驗的效果，一個網頁中不應有超過一個以上的main標籤

  1. <main>
    各種標籤及內文
    </main>

nav tag

nav內通常包含導覽列區域，其內容包含網頁內的連結或導向其他網頁的連結

  1. <nav>
      <ul>
        <li><a href="#">連結文字</a></li>
        <li><a href="#">連結文字</a></li>
      </ul>
    </nav>

p tag

p tag的語意為內文的段落，通常一個p內會有一段句子，同時p自帶display:block特性會強制占用一行空間

  1. <p>內文</p>
  2. <p>內文</p>
  3. <p>內文</p>

br tag

br的用意在於斷行，通常使用於文字有其他含意的地方。

  1. <p>內文內文<br>
    內文</p>

img tag

img為置入圖片的語法，必須包含src參數，可選用的參數有alt、title等

  1. <img src="圖片連結位置">

aside tag

aside的語意為側邊獨立區域，與article、section類似，通常用於側邊導覽列等

  1. <aside>
    其他標籤及內容
    </aside>

header tag

header的位置通常於區域內的頂部，其大多數內含標題或大綱等

  1. <header>標籤或內文</header>

footer tag

footer的位於通常於網頁底部或區域內底部，用於表達版權相關訊息或聯絡資訊等訊息

  1. <footer>標籤或內容</footer>

body tag

body的內容為會在畫面上呈現的元素，任何要顯示在畫面上的元素都必須放在body內

  1. <body>
    呈現在畫面上的標籤及內容
    </body>

html tag

html標籤是整個網頁檔案最外層的標籤，其內容包含一切網頁文檔標籤，允許在html外的標籤只有!doctype宣告檔案型別

  1. <!DOCTYPE html>
    <html lang="網頁語系如zh-TW">
      <head>
      </head>
      <body>
      </body>
    </html>

meta tag

meta是一種透過各種參數告訴搜尋引擎要顯示網站時呈現哪些元素或內容，此標籤用於SEO優化居多

  1. <meta charset="網頁使用的編碼如UTF-8">
  2. <meta name="description" content="搜尋引擎中顯示的網站內文" />
  3. <meta property="og:image" content="在FB上預覽網站要顯示的縮圖連結"/>

head tag

head位置位於網頁原始碼的頂部，其用意為包覆網頁提供搜尋引擎的訊息如:title、meta標籤以及導入css樣式連結也會在head標籤內

  1. <head>
      <meta 相關參數>
      <meta 相關參數>
      <title網站的標題</title>
      <link rel="stylesheet" href="css檔案位置">
    </head>

style tag

style位置依照開發者習慣會在網頁各個不同的地方，只有在style tag標籤內才可以使用css樣式

  1. <style>
    css相關樣式參數...
    </style>

link tag

link包含了許多參數，其用意為導入其他檔案至網頁中，最常見的如css樣式連結、標題icon圖示等

  1. <link rel="shortcut icon" href="顯示在網頁標題的圖片位置">
    <link rel="stylesheet" href="css檔案位置">

script tag

script內只能使用JS相關語法或搭配src參數將JS檔案導入網頁，隨著網頁內容越來越多，script的位置現在大多位於網頁底部</body>前，避免優先載入過多JS檔案而HTML、CSS載入緩慢導致使用者瀏覽體驗不佳的情況

  1. <script>
    js語法...
    </script>
  2. <script src="js檔案位置">

table tag

table為表格最外層的標籤，用於包覆與表格相關的標籤屬性，其子元素的參數有非常多如:thead、tbody、tr、th、td等

  1. <table>
      <thead>
        <tr>
          <th colspan="佔欄位格數如:2">這是標題</th>
        </tr>
     </thead>
      <tbody>
        <tr>
          <td>內文</td>
          <td>內文</td>
      </tbody>
    </table>

select tag

select為下拉式選單元件，其子元素必須要有option作為選項，建議使用參數value

  1. <select>
      <option>
        選項內容
      </option>
    </select>

input tag

input有非常多類型，使用type參數作為設定如:input type="text"等，若需要使用form表單功能input name參數則為強烈建議填入，name用於送出表單後後端根據name判斷格式

  1. <input type="類別" name="input的名稱" value="input的值">

小結

隨著網頁越來越多元化，如何正確使用HTML標籤建立結構不僅會影響編寫CSS的難易度也會對網站SEO有一定的影響。

要了解網頁結構是否有問題可以使用開發人員工具(F12)Elements的功能可以查看標籤對應的區塊。

更多關於網頁元素可以參考MDN HTML
